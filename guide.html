The Three Pillars of Omni-Channel

Understand the core components that power intelligent work distribution. Click each pillar to learn more.

Pillar 1: Presence - Managing Agent Availability

Presence is how agents signal their availability. It's the "on/off" switch for the work-flow, allowing you to control who gets what type of work, and when. Well-designed statuses like `Available - Chat` or `In Training` provide granular control and valuable data on how agents spend their time.

Pillar 2: Capacity - Defining Agent Workload

Capacity is the core feature that prevents agent overload. Each agent has a total capacity, and each work item (like a case or chat) consumes a portion of it. The system won't push more work than an agent can handle, ensuring balanced workloads and protecting service quality.

Pillar 3: Routing - The Engine of Distribution

Routing is the set of rules that directs work to its destination. It determines priority (which work gets handled first) and distribution model (which agent gets the next item). This is where you define the logic to match work to the best available agent based on queues, skills, or priority.
