The Three Pillars of Omni-Channel

1

Presence

The "On/Off" switch. Agents signal their availability (e.g., `Available - Chat`, `In Training`) to control the flow of work.

2

Capacity

The workload limit. Defines how many tasks an agent can handle at once to prevent overload and protect quality.

3

Routing

The engine. The rules that direct work based on priority, agent skills, and availability to find the perfect match.

Key Decision: Routing Strategy

Choose the right model based on your primary business goal: speed or precision.

Queue-Based (Simple & Fast)

Routes to the next available agent in a team. Best for generalist teams where speed is the top priority.

Skills-Based (Precise & Effective)

Matches work to agents with specific, verified skills. Best for specialized teams to maximize expertise.

Key Decision: Capacity Model

This choice is critical for accurate workload data. Use the right model for the right type of work.

Tab-Based Capacity

Best for real-time work like Chat

How it works: Capacity is consumed only when the work tab is open. Closing the tab releases capacity, even if the work isn't done. Risk: Creates hidden backlogs.

Status-Based Capacity

Best for long-running work like Cases

How it works: Capacity is tied to the work's actual status (e.g., 'In Progress'). Capacity is released when status is 'Waiting on Customer'. Benefit: Highly accurate data.

How It All Works Together

A simplified view of how a work item travels through the Omni-Channel system.

1. Work Item Arrives (Case, Chat, etc.)
↓
2. System Checks Priority & Required Skills
↓
3. Finds Agents who are Available + have matching Skills
↓
4. System Checks Agents' Remaining Capacity
↓
5. Pushes Work to the Best-Fit Agent!

Configurations in Action

See how different setups solve real business challenges.

High-Volume Retailer

Challenge: Urgent chats get stuck behind emails.

  • ✓ Use **Priority Routing** to handle chats first.
  • ✓ Make emails **Interruptible** to free up agents for chats.

Specialized Tech Team

Challenge: Complex cases go to junior agents.

  • ✓ Implement **Skills-Based Routing** for precision.
  • ✓ Use **Status-Based Capacity** for long, complex cases.

B2B Account Team

Challenge: AMs miss calls while doing case work.

  • ✓ Use **Direct-to-Agent** routing for key client calls.
  • ✓ **Status-Based Capacity** frees AMs when cases are paused.
